# 三、文档分层

ClinMindRuntime 文档按五层管理。

```text
第 1 层：总设计
  定义系统是什么、由哪些能力域组成、主链路如何运行。

第 2 层：项目设计地图 / 文档导航
  说明每份文档的归属、状态、用途和实现关系。

第 3 层：专项设计文档
  展开某个能力域或工程主题，例如 Agent、RAG、模型训练、Python Provider、数据安全、测试、部署等。

第 4 层：Phase 实现规格与任务清单
  说明某个阶段具体实现什么、不实现什么、如何验收。

第 5 层：实现约束 / AI 编码入口 / 冻结记录
  约束 AI 编码工具、记录当前阶段状态、保存阶段完成后的冻结事实。
```

推荐阅读顺序：


# 九、当前禁止事项

在 Phase 8-P0 规格未建立前，不应直接实现：

```text
1. Python AI Provider 代码。
2. Python Runtime 主控。
3. Python Agent 自主循环。
4. LLM 直接生成患者回答。
5. Embedding / reranker 直接改写 EvidenceGraph。
6. 模型训练 / 后训练流水线。
7. ModelRegistry / PromptRegistry 生产平台。
8. MCP / Tool / Skills 正式接入。
9. ExperienceCandidate 自动上线。
10. TrainingDatasetVersion 自动发布。
```
