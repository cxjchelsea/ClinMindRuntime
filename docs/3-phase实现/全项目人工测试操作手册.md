# 全项目人工测试操作手册：ClinMindRuntime

> 适用阶段：Phase 1–10 P0 当前整体闭环  
> 测试目标：验证 ClinMindRuntime 作为“受控医疗 AI Agent Runtime 与能力治理平台”的完整原型闭环是否可运行、可观察、可追踪、可审计、可评估，并确认各能力域没有越权。  
> 测试方式：浏览器 Console 页面 + Debug API / Postman / Apifox / curl。  
> 测试重点：不是验证真实医疗诊断能力，而是验证 Runtime 主控、Policy、Validation、Trace、Audit、Evaluation、Candidate、Console Observation 是否形成闭环。

---

# 一、测试总目标

本次完整人工测试要证明：

```text
Runtime 主控
-> Agent / Evidence / Provider / ModelGov / ToolGov
-> Policy / Validation
-> Trace / Audit
-> Evaluation / Candidate
-> Governance Console 可观察
```

其中每一层都必须满足：

```text
1. Agent 只能提出 Proposal，不能直接决定 RuntimeState。
2. RAG / Graph 只能提供 Evidence，不能直接生成患者结论。
3. Python Provider 只能供能，不能接管 Runtime。
4. Judge / Risk 只能提供辅助信号，不能替代 SafetyGate。
5. ModelRegistry / PromptRegistry / DatasetVersion 只登记和治理，不能自动发布。
6. Tool / MCP / Skills 只能返回结构化 ToolResult / ExternalContext / SkillResult，不能写 PatientOutput。
7. Console 只读观察，不能 approve / reject / publish / run。
8. 所有 Console 输出必须是 Safe DTO，不泄露 raw patient dialogue、raw prompt、secret、raw external response、chain-of-thought、full rationale。
```

```text

```

## 3.1 设置 API Base URL

操作：

```text
1. 找到左侧 Debug Context。
2. 找到输入框：API Base URL。
3. 如果前端通过 Vite 代理访问后端，则留空。
4. 如果不用代理，则输入：http://localhost:8080。
```

预期结果：

```text
页面顶部 API 显示：/api → localhost:8080 (proxy)
或显示你填写的 http://localhost:8080。
```

操作：

```text
1. 找到 X-Debug-Actor 输入框。
2. 输入：manual-tester。
```

预期结果：

```text
页面顶部 Actor 显示 manual-tester。
```

## 3.3 设置 Roles

操作：

```text
1. 找到 X-Debug-Roles。
2. 勾选 SYSTEM_ADMIN。
3. 可以取消 READ_ONLY_OBSERVER，也可以保留。
```

预期结果：

```text
页面顶部 Roles 包含 SYSTEM_ADMIN。
```

## 3.4 测试连接

操作：

```text
1. 点击 Debug Context 面板中的“测试连接”按钮。
```

预期结果：

```text
显示“连接成功”。
```

如果失败：

```text
1. 检查后端是否启动。
2. 检查 API Base URL 是否正确。
3. 检查 Debug Token 是否需要填写。
4. 检查浏览器控制台 Network 中 /api/v1/debug/console/runtime-sessions?limit=1 是否返回 200。
```

# 五、Debug API 人工测试

以下测试建议使用 Postman / Apifox / curl.exe。Windows PowerShell 中建议使用 `curl.exe`，避免和 PowerShell alias 冲突。

统一请求头：

```text
Content-Type: application/json
X-Debug-Actor: manual-tester
X-Debug-Roles: SYSTEM_ADMIN
X-Request-Id: manual-test-001
```

如果启用了 debug token，再加：

```text
X-Debug-Token: <你的 token>
```

---

## API-001：Console Overview API

### 请求

```bash
curl.exe -X GET "http://localhost:8080/api/v1/console/overview" ^
  -H "X-Debug-Actor: manual-tester" ^
  -H "X-Debug-Roles: SYSTEM_ADMIN" ^
  -H "X-Request-Id: manual-console-overview"
```

### 预期输出

HTTP：

```text
200 OK
```

响应结构：

```json
{
  "success": true,
  "data": {
    "runtime_count": 0,
    "provider_call_count": 0,
    "tool_invocation_count": 0,
    "model_governance_record_count": 0,
    "candidate_count": 0,
    "audit_event_count": 0,
    "domain_cards": []
  }
}
```

说明：具体数字可以不是 0，以实际数据为准。

验收点：

```text
1. success=true。
2. data 存在。
3. 不出现 raw patient dialogue / prompt_text / secret / raw_external_response。
```
